- **最小尺寸：** 44×44点（iOS） / 48×48dp（Android）

- **间距：** 目标之间最小间距为 8 像素

## iOS 人机界面指南（HIG）

### 设计哲学

- **清晰度：** 文字可读，图标精确，装饰元素细腻
- **谦逊：** 界面帮助用户理解内容，从不与内容竞争
- **深度：** 明确的视觉层次传达信息结构

### 导航模式

| 模式 | 使用场景 |
|------|----------|
| 标签栏 | 3-5 个顶层目的地 |
| 导航栏 | 分层内容 |
| 侧边栏 | iPad、内容丰富的应用 |
| 搜索 | 内容发现 |

### 标签栏规范

```text
┌─────────────────────────────────┐
│  🏠    🔍    ➕    💬    👤    │
│ Home  Search Add  Chat  Profile │ 49点高度
└─────────────────────────────────┘

```

- 最多 5 个标签
- 图标尺寸为 25×25 点，标签文字为 10 点
- 活动标签使用填充/色调颜色
- 非活动标签使用灰色

### 导航栏

```text
┌─────────────────────────────────┐
│ ‹ 返回    页面标题    操作 │ 44点最小
└─────────────────────────────────┘

```

- 左侧：返回按钮或取消按钮
- 中心：标题（可左对齐）
- 右侧：主要操作（文字或图标）

## Android 材料设计

### Android 设计哲学

- **材料作为隐喻：** 物理属性、层次感
- **大胆、图形化、有目的性：** 故意的色彩、字体、空间布局
- **运动赋予意义：** 反馈和连续性

### Android 导航模式

| 模式 | 使用场景 |
|------|----------|
| 底部导航 | 3-5 个顶层目的地 |
| 导航抽屉 | 5 个以上目的地，不常用 |
| 导航栏 | 平板电脑横屏 |
| 标签 | 相关内容组 |

### 底部导航

```text
┌─────────────────────────────────┐
│  🏠    🔍    📷    💬    👤    │
│ Home  Search Camera Chat Account│ 80dp 高度
└─────────────────────────────────┘

```

- 3-5 个目的地
- 图标尺寸为 24dp，标签文字为 12sp
- 活动状态：填充图标 + 主要颜色
- 非活动状态：轮廓图标 + 表面颜色

### 应用栏

```text
┌─────────────────────────────────┐
│ ≡  应用标题                🔍 │ 64dp 高度
└─────────────────────────────────┘

```

- 左侧：导航图标（菜单或返回）
- 中心：标题（可左对齐）
- 右侧：操作图标（最多 3 个）

### 浮动操作按钮（FAB）

- **尺寸：** 标准 56dp，迷你 40dp
- **位置：** 右下角，距离边缘 16dp
- **用途：** 仅主要操作
- **行为：** 可以在滚动时隐藏

## 响应式网页设计

### 断点

| 名称 | 宽度 | 典型设备 |
|------|------|----------|
| xs | <576px | 横屏手机 |
| sm | 576-767px | 竖屏手机 |
| md | 768-991px | 平板电脑 |
| lg | 992-1199px | 小型桌面 |
| xl | 1200-1399px | 桌面 |
| xxl | ≥1400px | 大型桌面 |

### 网格系统

- **列数：** 标准为 12 列网格
- **列间距：** 列之间为 16-24 像素
- **边距：** 移动设备为 16 像素，桌面设备为 64 像素
- **最大内容宽度：** 1200-1440 像素

### 响应式字体

```text
移动设备（基础）：
  正文：16像素
  H1：28-32像素
  H2：22-24像素
  
平板电脑：
  正文：16像素
  H1：32-40像素
  H2：24-28像素
  
桌面设备：
  正文：16-18像素
  H1：40-56像素
  H2：28-36像素

```

### 移动优先设计

1. 首先设计最小屏幕
2. 为更大屏幕添加复杂性
3. 内容优先级：什么是最关键的？
4. 性能：为移动设备优化
5. 以触摸操作为主的设计

### 响应式模式

| 模式 | 描述 |
|------|------|
| 堆叠 | 移动设备上列变为行 |
| 重排 | 内容根据优先级重新排列 |
| 展示 | 更大的尺寸显示更多内容 |
| 画布导航 | 移动设备上导航滑入 |
| 缩放 | 元素按比例缩放 |

## 桌面应用程序

### 窗口边框

```text
┌─────────────────────────────────────────┐
│ ● ● ●   应用标题              ─ □ ×  │ 标题栏
├────────┬────────────────────────────────┤
│ 侧边栏 │ 内容区域                   │
│        │                                │
│        │                                │
│        │                                │
│        ├────────────────────────────────┤
│        │ 状态栏                     │
└────────┴────────────────────────────────┘

```

### 键盘优先设计

- 所有操作均可通过键盘完成
- 可见的键盘快捷键
- 为 tab 顺序管理焦点
- 搜索/命令面板（Cmd/Ctrl+K）

### 悬停状态

桌面设备支持悬停（移动设备不支持）：

- 悬停时显示附加信息
- 点击前预览操作
- 图标按钮的悬停提示
- 悬停时显示下拉菜单

### 密集信息

桌面设备允许：

- 更小的触摸目标（最小 32 像素）
- 更多可见信息
- 复杂表格和数据网格
- 多列布局
- 并排比较

## 跨平台考虑因素

### 共享原则

- 一致的品牌识别
- 相同的核心用户流程
- 同步的数据/状态
- 熟悉的信息架构

### 平台特定适配

| 方面 | iOS | Android | Web |
|------|-----|---------|-----|
| 返回 | 左侧导航 | 左侧或手势 | 浏览器返回 |
| 主要操作 | 右侧导航 | FAB | 右上角按钮 |
| 列表 | 滑动手势操作 | 长按 | 悬停操作 |
| 菜单 | 操作面板 | 底部面板 | 下拉/上下文菜单 |
| 提示 | 居中模态 | 居中模态 | 不同位置 |

### 跨平台设计令牌

创建与平台无关的令牌：

```text
// 间距
spacing-sm: 8
spacing-md: 16
spacing-lg: 24

// 这些映射到平台单位
iOS: 点（pt）
Android: 密度无关像素（dp）
Web: 像素（px）或 rem

```
