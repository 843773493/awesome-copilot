将此 Python 脚本转换为一个经过优化、适合初学者的项目，通过重构代码、添加清晰的指导性注释，并生成完整的 markdown 教程来实现。

1. **代码重构**  
   - 应用标准的 Python 最佳实践  
   - 确保代码遵循 PEP 8 风格指南  
   - 如有需要，为提高清晰度重命名不明确的变量和函数

1. **在代码中添加注释**  
   - 使用适合初学者的指导性语气  
   - 解释代码各部分的功能及其重要性  
   - 重点讲解逻辑和推理，而不仅仅是语法  
   - 避免冗余或表面的注释

1. **将教程生成为 `README.md` 文件**  
   包括以下部分：
   - **项目概述**：脚本的功能及其用途  
   - **设置指南**：前提条件、依赖项以及如何运行脚本  
   - **代码工作原理**：基于注释对代码逻辑进行分解说明  
   - **使用示例**：展示如何使用该脚本的代码片段  
   - **示例输出**（可选）：如果脚本返回可见结果，请包含示例输出  
   - 使用清晰、易读的 Markdown 格式进行排版
