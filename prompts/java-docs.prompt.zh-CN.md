

# Java 文档（Javadoc）最佳实践

- 公共和受保护的成员应使用 Javadoc 注释进行文档说明。
- 建议对包私有和私有成员进行文档说明，尤其是当它们较为复杂或不够明确时。
- Javadoc 注释的第一句话是摘要描述。它应简明扼要地概述该方法的功能，并以句号结尾。
- 使用 `@param` 注释方法参数。描述以小写字母开头，不以句号结尾。
- 使用 `@return` 注释方法返回值。
- 使用 `@throws` 或 `@exception` 注释方法抛出的异常。
- 使用 `@see` 注释对其他类型或成员的引用。
- 使用 `{@inheritDoc}` 从基类或接口继承文档。
  - 除非有重大行为变更，否则应记录差异。
- 使用 `@param <T>` 注释泛型类型或方法中的类型参数。
- 使用 `{@code}` 注释内联代码片段。
- 使用 `<pre>{@code ... }</pre>` 注释代码块。
- 使用 `@since` 注释指示功能引入的时间（例如版本号）。
- 使用 `@version` 注释指定成员的版本。
- 使用 `@author` 注释指定代码的作者。
- 使用 `@deprecated` 注释标记成员为已弃用，并提供替代方案。
