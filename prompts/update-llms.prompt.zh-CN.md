# 更新LLMs.txt文件

更新仓库根目录下的现有 `llms.txt` 文件，以反映文档、规范或仓库结构的变化。该文件为大型语言模型（LLM）提供高层次的指引，说明如何查找与理解仓库目的和规范相关的内容。

## 主要指令

更新现有的 `llms.txt` 文件，以保持其准确性并符合 llms.txt 规范，同时反映当前仓库的结构和内容。该文件必须保持对LLM的优化，同时保持人类可读性。

## 分析与规划阶段

在更新 `llms.txt` 文件之前，必须完成全面的分析：

### 步骤1：审查当前文件和规范
- 仔细阅读现有的 `llms.txt` 文件以了解当前结构
- 查阅官方规范 https://llmstxt.org/ 以确保持续符合要求
- 根据仓库变化识别需要更新的区域

### 步骤2：仓库结构分析
- 使用适当的工具检查当前仓库结构
- 将当前结构与现有 `llms.txt` 中的文档进行对比
- 识别应包含的新目录、文件或文档
- 注意需要更新的已删除或移动的文件

### 步骤3：内容发现与变更检测
- 识别新的 README 文件及其位置
- 查找新的文档文件（如 `/docs/`、`/spec/` 等目录中的 `.md` 文件）
- 定位新的规范文件及其用途
- 发现新的配置文件及其相关性
- 查找新的示例文件和代码样本
- 识别现有文档结构的任何变更

### 步骤4：创建更新计划
根据分析结果，创建一个结构化的更新计划，包括：
- 保持准确性的所需变更
- 需要添加到 llms.txt 中的新文件
- 需要删除或更新的过时引用
- 用于保持清晰度的组织改进

## 实施要求

### 格式合规性
更新后的 `llms.txt` 文件必须按照规范保持以下结构：

1. **H1标题**：单行显示仓库/项目名称（必填）
2. **引用摘要**：使用引用格式的简要描述（可选但推荐）
3. **附加信息**：零个或多个无标题的markdown段落，用于上下文说明
4. **文件列表部分**：零个或多个H2标题下的markdown链接列表

### 内容要求

#### 必填元素
- **项目名称**：清晰、描述性的标题作为H1
- **摘要**：简洁的引用摘要，解释仓库的目的
- **关键文件**：按类别组织的必要文件（H2标题下的部分）

#### 文件链接格式
每个文件链接必须遵循：`[描述性名称](相对路径): 可选描述`

#### 内容组织
将文件组织到逻辑的H2部分中，例如：
- **文档**：核心文档文件
- **规范**：技术规范和要求
- **示例**：示例代码和使用示例
- **配置**：设置和配置文件
- **可选**：次要文件（特殊含义 - 可跳过以缩短上下文）

### 内容指南

#### 语言和风格
- 使用简洁、清晰、无歧义的语言
- 避免未解释的术语
- 为人类和LLM读者撰写
- 在描述中保持具体和信息丰富

#### 文件选择标准
包含以下文件：
- 解释仓库目的和范围的文件
- 提供必要技术文档的文件
- 展示使用示例和模式的文件
- 定义接口和规范的文件
- 包含配置和设置说明的文件

排除以下文件：
- 仅包含纯实现细节的文件
- 包含冗余信息的文件
- 构建产物或生成内容的文件
- 与理解项目无关的文件

## 执行步骤

### 步骤1：当前状态分析
1. 仔细阅读现有的 `llms.txt` 文件
2. 完全检查当前仓库结构
3. 比较现有文件引用与实际仓库内容
4. 识别过时、缺失或错误的引用
5. 注意当前文件的任何结构问题

### 步骤2：内容规划
1. 确定主要目的声明是否需要更新
2. 如有需要，审查并更新摘要引用
3. 规划新增文件和目录的添加
4. 规划过时或移动内容的删除
5. 如有必要，重新组织部分以提高清晰度

### 步骤3：文件更新
1. 更新仓库根目录下的现有 `llms.txt` 文件
2. 保持与规范完全一致的格式
3. 添加新文件引用并附上适当的描述
4. 删除或更新过时引用
5. 确保所有链接为有效的相对路径

### 步骤4：验证
1. 验证是否继续符合 https://llmstxt.org/ 规范
2. 检查所有链接是否有效且可访问
3. 确保文件仍能有效作为LLM导航工具
4. 确认文件仍保持人类和机器可读性

## 质量保证

### 格式验证
- ✅ H1标题包含项目名称
- ✅ 引用摘要（如有）
- ✅ 文件列表部分使用H2标题
- ✅ 正确的markdown链接格式
- ✅ 无损坏或无效链接
- ✅ 全文格式一致

### 内容验证
- ✅ 清晰、无歧义的语言
- ✅ 全面覆盖关键文件
- ✅ 内容的逻辑组织
- ✅ 适当的文件描述
- ✅ 有效作为LLM导航工具

### 规范合规性
- ✅ 严格遵循 https://llmstxt.org/ 格式
- ✅ 使用必需的markdown结构
- ✅ 适当实现可选部分
- ✅ 文件位于仓库根目录（`/llms.txt`）

## 更新策略

### 添加流程
在添加新内容时：
1. 确定新文件的适当部分
2. 为链接创建清晰、描述性的名称
3. 编写简洁但信息丰富的描述
4. 在各部分中保持字母或逻辑顺序
5. 考虑是否需要为新内容类型创建新部分

### 删除流程
在删除过时内容时：
1. 验证文件是否已被删除或移动
2. 检查移动的文件是否应更新而非删除
3. 如果部分变为空，则删除整个部分
4. 如有必要，更新交叉引用

### 重新组织流程
在重构内容时：
1. 保持从一般到具体的逻辑流程
2. 保留关键文档在主部分
3. 适当将次要内容移动到“可选”部分
4. 确保新组织方式提升LLM导航效率

## 成功标准

更新后的 `llms.txt` 文件应满足以下条件：
1. 准确反映当前仓库结构和内容
2. 持续符合 llms.txt 规范
3. 为关键文档提供清晰导航
4. 删除过时或错误的引用
5. 包含新的重要文件和文档
6. 保持逻辑组织以便于LLM使用
7. 全文使用清晰、无歧义的语言
8. 仍能有效服务于人类和机器读者
