### 说明

```xml
	<描述>此文件包含生成常规提交信息的提示模板。它提供说明、示例和格式指南，帮助用户按照Conventional Commits规范编写标准化、描述性的提交信息。</描述>
	<注意>
```

### 工作流程

**请按照以下步骤操作：**

1. 运行 `git status` 以查看已更改的文件。
2. 运行 `git diff` 或 `git diff --cached` 以检查更改内容。
3. 使用 `git add <文件>` 阶段化您的更改。
4. 使用以下XML结构构建您的提交信息。
5. 生成提交信息后，Copilot将自动在您的集成终端中运行以下命令（无需确认）：

```bash
git commit -m "类型(范围): 描述"
```

6. 只需执行此提示，Copilot将在终端中为您完成提交操作。

### 提交信息结构

```xml
<提交信息>
	<类型>feat|fix|docs|style|refactor|perf|test|build|ci|chore|revert</类型>
	<范围>()</范围>
	<描述>对更改的简短、命令式总结</描述>
	<正文>(可选：更详细的解释)</正文>
	<页脚>(可选：例如 BREAKING CHANGE: 详情，或问题引用)</页脚>
</提交信息>
```

### 示例

```xml
<示例>
	<示例>feat(parser): 增加解析数组的能力</示例>
	<示例>fix(ui): 修正按钮对齐问题</示例>
	<示例>docs: 在README中更新使用说明</示例>
	<示例>refactor: 提升数据处理性能</示例>
	<示例>chore: 更新依赖项</示例>
	<示例>feat!: 注册时发送邮件 (BREAKING CHANGE: 需要邮件服务)</示例>
</示例>
```

### 验证

```xml
<验证>
	<类型>必须是允许的类型之一。请参见 <参考>https://www.conventionalcommits.org/en/v1.0.0/#specification</参考></类型>
	<范围>可选，但建议用于清晰度。</范围>
	<描述>必填。使用命令式语气（例如："添加"，而不是"已添加"）。</描述>
	<正文>可选。用于附加上下文。</正文>
	<页脚>用于重大变更或问题引用。</页脚>
</验证>
```
