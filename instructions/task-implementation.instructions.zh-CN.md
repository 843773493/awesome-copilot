# 任务计划实施指南

您将在 `.copilot-tracking/plans/**` 和 `.copilot-tracking/details/**` 中实施特定的任务计划。您的目标是按步骤逐步、完整地实现计划文件中的每个步骤，以创建高质量、可运行的软件，满足所有指定需求。

实施进度**必须**记录在位于 `.copilot-tracking/changes/**` 的对应变更文件中。

## 核心实施流程

### 1. 计划分析与准备

**必须在开始实施前完成：**
- **强制性**：阅读并完全理解完整的计划文件，包括范围、目标、所有阶段和每个检查清单项
- **强制性**：阅读并完全理解对应的变更文件 - 如果上下文缺失，使用 `read_file` 重新读取整个文件
- **强制性**：识别计划中提及的所有引用文件，并检查其上下文
- **强制性**：理解当前项目结构和规范

### 2. 系统化实施流程

**按计划系统化地实施每个任务：**

1. **按顺序处理任务** - 严格按照计划顺序逐项执行任务
2. **在实施任何任务前必须：**
   - **始终确保实施与计划中的具体任务相关联**
   - **始终从 `.copilot-tracking/details/**` 中关联的详细 Markdown 文件中读取该任务的完整详细信息部分**
   - **在继续之前，完全理解所有实施细节**
   - 按需收集任何额外的必要上下文

3. **使用可运行代码完整实施任务：**
   - 遵循工作区中已有的代码模式和规范
   - 创建符合详细信息中所有任务要求的可运行功能
   - 包含适当的错误处理、文档说明并遵循最佳实践

4. **标记任务完成并更新变更跟踪：**
   - 更新计划文件：将已完成任务的 `[ ]` 改为 `[x]`
   - **在完成每个任务后必须**：通过在 Added、Modified 或 Removed 部分添加相对文件路径和一句实施摘要，更新变更文件
   - **强制性**：如果任何变更偏离了任务计划和详细信息，请在相关部分明确指出该变更是在计划之外进行的，并包含具体原因
   - 如果某一阶段的所有任务均完成 `[x]`，则将该阶段标题标记为完成 `[x]`

### 3. 实施质量标准

**每次实施必须：**
- 遵循工作区已有的模式和规范（请检查 `copilot/` 文件夹以获取标准）
- 实现完整、可运行的功能，满足所有任务需求
- 包含适当的错误处理和验证
- 使用与工作区一致的命名规范和代码结构
- 对复杂逻辑添加必要的文档和注释
- 确保与现有系统和依赖项的兼容性

### 4. 持续进度与验证

**在实施每个任务后：**
1. 将所做的变更与详细文件中的任务需求进行验证
2. 在进入下一个任务前修复任何问题
3. **强制性**：更新计划文件以标记已完成的任务 `[x]`
4. **在完成每个任务后必须**：通过在 Added、Modified 或 Removed 部分添加相对文件路径和一句实施摘要，更新变更文件
5. 继续执行下一个未检查的任务

**持续直到：**
- 所有计划任务均标记为完成 `[x]`
- 所有指定文件已创建或更新，并包含可运行代码
- 所有计划中的成功标准均已验证

### 5. 参考资料收集指南

**在收集外部参考资料时：**
- 优先关注实际实施示例而非理论文档
- 验证外部来源是否包含实际可用的模式
- 将外部模式调整为符合工作区规范和标准

**在从参考资料实施时：**
- 首先遵循工作区的模式和规范，其次为外部模式
- 实现完整、可运行的功能，而非仅仅示例
- 确保所有依赖项和配置正确集成
- 确保实施在现有项目结构内正常运行

### 6. 完成与文档

**当实施完成时：**
- 所有计划任务均标记为完成 `[x]`
- 所有指定文件均存在且包含可运行代码
- 所有计划中的成功标准均已验证
- 没有剩余的实施错误

**最终步骤 - 用发布摘要更新变更文件：**
- 仅在所有阶段均标记为完成 `[x]` 后，才在变更文件中添加最终的 Release Summary 部分
- 为发布文档记录完整的文件清单和整体实施摘要

### 7. 问题解决

**在遇到实施问题时：**
- 清晰记录具体问题
- 尝试替代方法或搜索其他关键词
- 在外部参考资料失败时，使用工作区模式作为备用方案
- 在可用信息的基础上继续实施，而非完全停止
- 在计划文件中记录任何未解决的问题，供将来参考

## 实施流程

```
1. 阅读并完全理解计划文件和所有检查清单
2. 阅读并完全理解变更文件（如果上下文缺失，请重新读取整个文件）
3. 对于每个未检查的任务：
   a. 从详细 Markdown 文件中读取该任务的完整详细信息部分
   b. 完全理解所有实施需求
   c. 按照工作区模式实现任务的可运行代码
   d. 验证实施是否符合任务需求
   e. 在计划文件中标记任务为完成 [x]
   f. 在变更文件的 Added、Modified 或 Removed 部分添加条目
   g. 在相关部分明确指出与计划/详细信息的任何偏离，并包含具体原因
4. 重复此流程直至所有任务完成
5. 仅在所有阶段均完成 [x] 后：在变更文件中添加最终的发布摘要
```

## 成功标准

当满足以下条件时，实施视为完成：
- ✅ 所有计划任务均标记为完成 `[x]`
- ✅ 所有指定文件包含可运行代码
- ✅ 代码遵循工作区模式和规范
- ✅ 所有功能在项目中按预期运行
- ✅ 每完成一个任务后，变更文件均通过 Added、Modified 或 Removed 条目更新
- ✅ 变更文件记录所有阶段并包含详细的发布就绪文档和最终发布摘要
