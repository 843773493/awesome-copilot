# Arch Linux 管理指南

在编写 Arch Linux 系统的指导说明、脚本或文档时，请使用以下指令。

## 平台对齐

- 强调滚动发行版模型以及进行完整升级的必要性。
- 在排查问题时，确认当前内核和近期软件包的变更。
- 优先使用官方仓库和 Arch Wiki 获取权威指导。

## 包管理

- 使用 `pacman -Syu` 进行完整系统升级；避免部分升级。
- 使用 `pacman -Qi`、`pacman -Ql` 和 `pacman -Ss` 检查软件包。
- 仅在明确警告和 PKGBUILD 审核提醒的情况下提及 AUR 辅助工具。

## 配置与服务

- 将配置文件保留在 `/etc` 目录下，避免编辑 `/usr` 目录中的文件。
- 使用位于 `/etc/systemd/system/<unit>.d/` 的 systemd drop-in 配置文件。
- 使用 `systemctl` 和 `journalctl` 进行服务控制和日志管理。

## 安全

- 注意在内核或核心库升级后需要重启系统。
- 建议遵循最小特权原则使用 sudo，并保持软件包数量最少。
- 明确指出防火墙工具的预期配置（如 nftables/ufw）。

## 交付物

- 以可直接复制粘贴的代码块形式提供命令。
- 在更改后包含验证步骤。
- 为高风险操作提供回滚或清理步骤。
