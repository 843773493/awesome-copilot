# Next.js + Tailwind 开发指南

关于使用 Tailwind CSS 样式和 TypeScript 开发高质量 Next.js 应用的指导方针。

## 项目背景

- 最新的 Next.js（应用路由）
- 使用 TypeScript 实现类型安全
- 使用 Tailwind CSS 进行样式设计

## 开发标准

### 架构
- 应用路由与服务端和客户端组件
- 按功能/领域分组路由
- 实现适当的错误边界
- 默认使用 React 服务端组件
- 在可能的情况下利用静态优化

### TypeScript
- 启用严格模式
- 明确的类型定义
- 使用类型守卫进行适当的错误处理
- 使用 Zod 进行运行时类型验证

### 样式设计
- 使用 Tailwind CSS 并保持一致的颜色调色板
- 响应式设计模式
- 暗模式支持
- 遵循容器查询最佳实践
- 保持语义化 HTML 结构

### 状态管理
- 使用 React 服务端组件处理服务端状态
- 使用 React Hook 处理客户端状态
- 实现适当的数据加载和错误状态
- 在合适的情况下使用乐观更新

### 数据获取
- 使用服务端组件直接查询数据库
- 使用 React 挂起处理加载状态
- 实现适当的错误处理和重试逻辑
- 缓存失效策略

### 安全性
- 输入验证和清理
- 正确的认证检查
- CSRF 防护
- 速率限制的实现
- 安全的 API 路由处理

### 性能优化
- 使用 next/image 进行图片优化
- 使用 next/font 进行字体优化
- 路由预加载
- 正确的代码分割
- 打包体积优化

## 实现流程
1. 规划组件层级结构
2. 定义类型和接口
3. 实现服务端逻辑
4. 构建客户端组件
5. 添加适当的错误处理
6. 实现响应式样式
7. 添加加载状态
8. 编写测试
