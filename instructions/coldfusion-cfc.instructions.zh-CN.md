# CFC 文件的 ColdFusion 编码规范

- 尽可能使用 CFScript 以获得更清晰的语法。
- 避免使用已弃用的标签和函数。
- 遵循变量和组件的一致命名约定。
- 使用 `cfqueryparam` 来防止 SQL 注入。
- 在 <cfoutput> 块中使用 `##` 转义 CSS 哈希符号。

# 其他最佳实践

- 在适当的情况下，使用 `this` 作用域来定义组件属性和方法。
- 为所有函数记录其用途、参数和返回值（使用 Javadoc 或类似风格）。
- 为函数和变量使用访问修饰符（`public`, `private`, `package`, `remote`）。
- 优先使用依赖注入进行组件协作。
- 避免在 setters/getters 中包含业务逻辑；保持它们简单。
- 验证和清理 public/remote 方法中的所有输入参数。
- 根据需要在方法内部使用 `cftry`/`cfcatch` 进行错误处理。
- 避免在 CFC 中硬编码配置或凭证。
- 使用一致的缩进（2 个空格，符合全局标准）。
- 在组件内逻辑地分组相关方法。
- 为方法和属性使用有意义且描述性的名称。
- 避免使用已弃用或不必要的 `cfcomponent` 属性。

- 尽可能使用三元运算符
- 确保一致的制表符对齐。
