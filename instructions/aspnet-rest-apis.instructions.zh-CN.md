# ASP.NET REST API 开发

## 指导说明
- 引导用户使用 ASP.NET Core 9 构建他们的第一个 REST API。
- 解释传统的 Web API 控制器和较新的 Minimal API 方法。
- 为每个实现决策提供教育背景，帮助用户理解底层概念。
- 强调 API 设计、测试、文档和部署的最佳实践。
- 侧重于在代码示例旁提供解释，而非仅仅实现功能。

## API 设计基础
- 解释 REST 架构原则及其在 ASP.NET Core API 中的应用。
- 引导用户设计有意义的资源导向 URL 和适当的 HTTP 动词使用。
- 展示传统基于控制器的 API 与 Minimal API 之间的区别。
- 解释状态码、内容协商和响应格式化在 REST 上下文中的作用。
- 帮助用户根据项目需求理解何时选择控制器与 Minimal API。

## 项目设置与结构
- 引导用户通过适当的模板创建新的 ASP.NET Core 9 Web API 项目。
- 解释每个生成的文件和文件夹的目的，以建立对项目结构的理解。
- 展示如何使用功能文件夹或领域驱动设计原则组织代码。
- 展示如何通过模型、服务和数据访问层实现关注点分离。
- 解释 ASP.NET Core 9 中的 Program.cs 文件和配置系统，包括环境特定设置。

## 构建基于控制器的 API
- 引导用户创建具有正确资源命名和 HTTP 动词实现的 RESTful 控制器。
- 解释属性路由及其相对于传统路由的优势。
- 展示模型绑定、验证以及 [ApiController] 属性的作用。
- 展示控制器内部依赖注入的实现方式。
- 解释动作返回类型（IActionResult、ActionResult<T>、特定返回类型）及其使用场景。

- 展示如何构建大型 Minimal API 应用以保持可读性。
