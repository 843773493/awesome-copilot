# HTML/CSS 颜色样式指南

在更新或创建HTML/CSS样式以实现浏览器渲染时，请遵循以下指南。颜色名称代表其对应色系的完整光谱（例如，“blue”包括海军蓝、天蓝色等）。

## 颜色定义

- **暖色系**：橙色、红色和黄色
- **冷色系**：蓝色、绿色和紫色
- **中性色**：灰色和灰度变体
- **二元色**：黑色和白色
- **60-30-10 规则**
  - **主色**：使用60%（冷色或浅色）
  - **辅色**：使用30%（冷色或浅色）
  - **点缀色**：使用10%（互补的暖色）

## 颜色使用指南

通过应用 **60-30-10 规则** 来平衡图形设计元素中使用的颜色，例如背景、按钮、卡片等。

### 背景色

**不要使用：**

- 紫色或品红
- 红色、橙色或黄色
- 粉红色
- 任何暖色

**推荐使用：**

- 白色或浅白色
- 浅冷色（例如浅蓝色、浅绿色）
- 微妙的中性色调
- 颜色变化极少的浅渐变

### 文字颜色

**不要使用：**

- 黄色（对比度和可读性差）
- 粉红色
- 在浅色背景上使用纯白色或浅色文字
- 在深色背景上使用纯黑色或深色文字

**推荐使用：**

- 深中性色（例如 #1f2328, #24292f）
- 接近黑色的变体（#000000 到 #333333）
  - 确保背景为浅色
- 深灰色（#4d4d4d, #6c757d）
- 无障碍设计的高对比度组合
- 接近白色的变体（#ffffff 到 #f0f2f3）
  - 确保背景为深色

### 应避免的颜色

除非设计规范或用户明确要求，否则应避免使用以下颜色：

- 鲜艳的紫色和品红
- 鲜艳的粉红色和霓虹色
- 高饱和度的暖色
- 对比度比率低的颜色（不符合WCAG可访问性标准）

### 应谨慎使用颜色

**暖色系**（红色、橙色、黄色）：

- 仅用于关键警报、警告或错误信息
- 仅在传达紧急情况或重要性时使用
- 限制在小面积的点缀色中，而非大面积区域
- 在使用暖色系之前，考虑使用图标或粗体文字作为替代方案

## 渐变色

使用微妙的颜色过渡来应用渐变色，以保持专业外观。

### 最佳实践

- 保持颜色变化极少（例如 #E6F2FF 到 #F5F7FA）
- 在同一色系内使用渐变色
- 避免在单一渐变色中混合暖色和冷色
- 背景优先使用线性渐变而非径向渐变

### 合适的使用场景

- 背景容器和区域
- 按钮悬停状态和交互元素
- 阴影和深度效果
- 标题栏和导航栏
- 卡片组件和面板
