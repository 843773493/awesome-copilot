# Python 编码规范

## Python 编码指导

- 为每个函数编写清晰简洁的注释。
- 确保函数具有描述性的名称并包含类型提示。
- 按照 PEP 257 规范提供文档字符串。
- 使用 `typing` 模块进行类型注解（例如 `List[str]`、`Dict[str, int]`）。
- 将复杂的函数分解为更小、更易管理的函数。

## 通用编码指导

- 始终优先考虑代码的可读性和清晰度。
- 对于涉及算法的代码，包含所采用方法的解释。
- 编写具有良好可维护性实践的代码，包括对某些设计决策的注释说明。
- 处理边界情况并编写清晰的异常处理。
- 对于库或外部依赖项，在注释中说明其用途和使用方式。
- 使用一致的命名约定并遵循语言特定的最佳实践。
- 编写简洁、高效且符合 Python 习惯的代码，同时确保易于理解。

## 代码风格与格式

- 遵循 **PEP 8** 的 Python 风格指南。
- 保持正确的缩进（每级缩进使用4个空格）。
- 确保行长度不超过79个字符。
- 在 `def` 或 `class` 关键字之后立即放置函数和类的文档字符串。
- 适当使用空行分隔函数、类和代码块。

## 边界情况与测试

- 始终为应用程序的关键路径编写测试用例。
- 考虑常见边界情况，如空输入、无效数据类型和大型数据集。
- 为边界情况及其预期行为添加注释。
- 为函数编写单元测试，并使用文档字符串解释测试用例。

## 正确文档的示例

```python
def calculate_area(radius: float) -> float:
    """
    根据半径计算圆的面积。
    
    参数:
    radius (float): 圆的半径。
    
    返回:
    float: 圆的面积，计算公式为 π 乘以半径的平方。
    """
    import math
    return math.pi * radius ** 2
```
