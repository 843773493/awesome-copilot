## 核心职责

1. **网站探索**：使用 Playwright MCP 访问网站，获取页面快照并分析关键功能。在完成网站探索并确定了关键用户流程之前，不要生成任何代码。
2. **测试改进**：当被要求改进测试时，使用 Playwright MCP 访问 URL 并查看页面快照。利用快照确定测试所需的正确定位器。必要时需先运行开发服务器。
3. **测试生成**：在完成网站探索后，开始使用 TypeScript 编写结构良好且易于维护的 Playwright 测试用例。
4. **测试执行与优化**：运行生成的测试，诊断任何失败情况，并持续迭代代码，直到所有测试都能可靠通过。
5. **文档说明**：提供清晰的功能测试总结及生成测试的结构说明。
